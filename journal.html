Покрытие API тестами: Postman и автопроверки

Покрыли все основные эндпоинты API автотестами в Postman: создание, обновление, публикация пакетов, версии, теги и авторизация.

Читать далее
  • POST /register — регистрация нового пользователя.
  • POST /login — вход по существующему аккаунту.
  • POST /token — обновление access токена.
  • POST /packages — создание пакета.
  • GET /packages/:id — получить пакет по ID.
  • GET /packages/:slug — получить пакет по slug.
  • PUT /packages — обновить пакет.
  • GET /packages/:id/updated — получить обновлённый пакет.
  • POST /packages/:id/version — добавить новую версию.
  • GET /packages/:id/versions — список версий.
  • GET /versions/:id — получить контент версии.
  • POST /packages/:id/publish — публикация без прав (проверка).
  • POST /packages/:id/tags — привязка тегов.
  • GET /tags — получить список тегов.
  • DELETE /tags/:id — удалить тег.
  • POST /admin/login — вход админа.
  • POST /admin/publish — публикация от имени админа.
  • GET /packages/search — поиск пакетов по фильтрам.

Каждый тест проверяет статус 200, валидность ID и slug, и соответствие переменным окружения. Это ускоряет разработку и предотвращает регрессии.

Обработка файлов PDF и DOCX

Реализована универсальная функция обработки различных типов файлов: PDF, DOCX, изображений и таблиц. Использованы популярные Python-библиотеки.

Читать далее

Используемые библиотеки:

  • pdfplumber — для обработки PDF-файлов (текст, таблицы, изображения).
  • python-docx — для извлечения текста, таблиц и изображений из DOCX.
  • csv, StringIO, BytesIO — для конвертации таблиц в формат CSV.
  • base64 — для кодирования изображений и вставки их в HTML.

Алгоритм работы функции:

  1. Определение типа файла:
    • Если PDF — используется pdfplumber.
    • Если DOCX — используется python-docx.
  2. Обработка PDF:
    • Проход по всем страницам.
    • Извлечение текста → оборачивается в <p> и разбивается на абзацы.
    • Извлечение таблиц → конвертация в CSV → вывод в <pre>.
    • Извлечение изображений → кодировка в base64 → вставка в <img>.
  3. Обработка DOCX:
    • Текст: каждый непустой абзац оборачивается в <p>.
    • Таблицы: преобразование в CSV → отображение в <pre>.
    • Изображения: кодируются в base64 и вставляются как <img>.

В результате формируется полноценная HTML-строка, отражающая структуру исходного документа: текст, таблицы и изображения в удобном формате.

Успешное прохождение второй аттестации проекта EasyAccess

Команда проекта EasyAccess рада сообщить о успешном прохождении второй аттестации. На предзащите мы продемонстрировали ключевые достижения и функционал расширения, получив высшую оценку комиссии.

Что было представлено на аттестации?

  1. Редактор сайтов:
    • Инструмент для создания и настройки адаптивных сценариев под конкретные веб-ресурсы.
    • Возможность тестирования изменений в реальном времени.
  2. Платформа с готовыми пакетами:
    • Библиотека предустановленных решений для популярных сайтов (соцсети, новостные порталы, образовательные платформы).
    • Фильтрация пакетов по категориям: "Для слабовидящих", "Для слабослышащих", "Упрощенный интерфейс".
  3. Экспорт и импорт настроек:
    • Пользователи могут сохранять свои конфигурации в формате JSON и делиться ими с сообществом.
    • Загрузка сторонних пакетов через встроенный маркетплейс расширения.
  4. Демонстрация работы приложения:
    • Показаны кейсы адаптации сайтов: изменение цветовых схем, динамическое увеличение текста, синхронизация субтитров с видео.
    • Пример интеграции с YouTube, Wikipedia и онлайн-библиотеками.
